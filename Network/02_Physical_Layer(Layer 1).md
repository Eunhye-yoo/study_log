# Physical Layer (Layer 1)

## 개요

Physical Layer는 OSI 7계층의 가장 하위 계층으로, 데이터를 실제로 전송하는 계층이다.

컴퓨터가 사용하는 데이터는 결국 0과 1(Bit)이며, 이를 전기 신호 또는 빛 신호로 변환하여 케이블을 통해 전달한다.

> 쉽게 말해, 네트워크에서 실제 "길"과 "신호"를 담당하는 계층이다.

- PDU : Bit

---

## 전송 매체 (Media)

전송 매체는 장비와 장비를 연결하여 데이터를 전달하는 통신 경로 역할을 한다.

예를 들어 인터넷을 사용하기 위해서는 컴퓨터와 공유기가 연결되어야 하는데, 이때 케이블이나 Wi-Fi가 전송 매체가 된다.

### Twisted Pair Cable

구리선을 서로 꼬아서 만든 케이블이다.

랜선이라고 하면 대부분 이 케이블을 의미한다.

#### UTP (Unshielded Twisted Pair)

- 차폐막(보호막)이 없음
- 가격이 저렴함
- 가정 및 사무실에서 가장 많이 사용
- 최대 전송 거리 약 100m

#### STP (Shielded Twisted Pair)

- 차폐막이 있음
- 전자기 간섭에 강함
- 공장, 서버실 등에서 사용

> 왜 선을 꼬아 놓을까?
>
> 전선 주변에는 전자기장이 발생하는데,
> 선을 꼬아 놓으면 서로 발생시키는 잡음을 상쇄하여 통신 품질이 좋아진다.

---

### Coaxial Cable

동축 케이블이라고 한다.

중심 도선을 절연체와 금속 차폐층으로 감싼 구조이다.

특징

- 외부 간섭에 강함
- 케이블 TV 등에 사용
- 현재 LAN 환경에서는 거의 사용하지 않음

---

### Fiber Optic Cable

광섬유 케이블이라고 한다.

전기가 아닌 빛을 이용하여 데이터를 전송한다.

특징

- 매우 빠른 속도
- 장거리 전송 가능
- 전자기 간섭의 영향을 받지 않음

사용 예

- 통신사 백본망
- 데이터센터 연결
- 해저 광케이블

> 우리가 해외 서버에 접속할 때도 많은 데이터가 해저 광케이블을 통해 이동한다.

---

## Repeater

### 왜 필요한가?

전기 신호는 멀리 갈수록 점점 약해진다.

사람이 큰 소리로 외쳐도 멀리 가면 잘 들리지 않는 것과 비슷하다.

Repeater는 약해진 신호를 다시 증폭하여 더 멀리 전달할 수 있도록 도와주는 장비이다.

### 특징

- Layer 1 장비
- 신호를 재생 및 증폭
- 데이터를 분석하지 않음

---

## Hub

### 왜 필요한가?

컴퓨터가 여러 대라면 서로 연결할 장비가 필요하다.

Hub는 여러 장비를 하나의 네트워크에 연결하기 위해 사용되는 장비이다.

### 동작 방식

Hub는 목적지를 구분할 수 없다.

따라서 데이터를 수신하면 입력된 포트를 제외한 모든 포트로 데이터를 전달한다.

이를 Flooding이라고 한다.

예)

PC-A → 데이터 전송

↓

Hub 수신

↓

PC-B, PC-C, PC-D 모두에게 전달

> Hub는 "누가 받을지 모르니까 일단 전부 보내자" 방식으로 동작한다.

---

### Collision (충돌)

만약 PC-A 와 PC-B 가 동시에 데이터를 보내면 신호가 섞여버릴 수 있다.

이를 Collision이라고 한다.

Collision이 발생하면 데이터를 다시 보내야 하므로 네트워크 성능이 저하된다.

---

### 특징

- Layer 1 장비
- MAC Address를 모름
- 데이터를 분석하지 않음
- 반이중(Half Duplex) 통신

---

### 장점

- 구조가 단순함
- 가격이 저렴함

### 단점

- 불필요한 트래픽 발생
- Collision 발생
- 네트워크 성능 저하
- 보안성이 낮음

---

## 정리

- Physical Layer는 실제 신호를 전송하는 계층이다.
- PDU는 Bit이다.
- 케이블은 네트워크 장비를 연결하는 통신 경로 역할을 한다.
- Repeater는 약해진 신호를 증폭한다.
- Hub는 목적지를 구분하지 못하고 모든 포트로 데이터를 전달한다.
- Hub의 Collision 문제를 해결하기 위해 Switch가 등장하였다.
