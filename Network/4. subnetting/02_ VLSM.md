# VLSM (Variable Length Subnet Mask)

## VLSM이란?

VLSM(Variable Length Subnet Mask)은

하나의 네트워크를 서로 다른 크기의 Subnet으로 나누는 기술이다.

일반 Subnetting은

```text
모든 Subnet의 크기가 동일
```

하지만

VLSM은

```text
필요한 만큼만 나누기 가능
```

하다.

---

## 왜 VLSM을 사용할까?

예)

회사에 3개의 부서가 있다고 가정

```text
영업팀 : 100명

개발팀 : 50명

인사팀 : 10명
```

일반 Subnetting을 사용하면

모든 부서에 동일한 크기의 네트워크를 할당한다.

결과적으로

```text
인사팀은 10명만 사용하는데

100명 크기의 네트워크를 받게 됨
```

즉

IP Address가 낭비된다.

---

VLSM은

```text
100명 → 큰 네트워크

50명 → 중간 네트워크

10명 → 작은 네트워크
```

로 할당한다.

따라서

```text
IP 낭비 최소화
```

가 가능하다.

---

## VLSM의 핵심

필요한 Host 수에 맞게

Subnet Mask를 다르게 사용한다.

예)

```text
영업팀

192.168.1.0/25
```

```text
개발팀

192.168.1.128/26
```

```text
인사팀

192.168.1.192/28
```

각 팀이 서로 다른 Mask를 사용한다.

---

## VLSM 문제 풀이 순서

### 1단계

필요한 Host 수 확인

예)

```text
영업팀 : 100명

개발팀 : 50명

인사팀 : 10명
```

---

### 2단계

Host 수가 큰 네트워크부터 정렬

```text
100

↓

50

↓

10
```

반드시 큰 것부터 나눈다.

---

## 왜 큰 것부터 나눌까?

작은 것부터 나누면

큰 네트워크가 들어갈 공간이 부족해질 수 있다.

예)

```text
/28

/28

/28

/28
```

처럼 잘게 쪼개버리면

나중에

```text
Host 100개
```

가 필요한 네트워크를 만들 수 없다.

그래서

```text
큰 것부터
```

가 원칙이다.

---

### 3단계

Host Bit 계산

공식

```text
2^n - 2
```

---

예)

Host 100개

```text
2^6 - 2 = 62

부족
```

```text
2^7 - 2 = 126

가능
```

따라서

```text
Host Bit = 7
```

---

Host 50개

```text
2^5 - 2 = 30

부족
```

```text
2^6 - 2 = 62

가능
```

---

Host 10개

```text
2^3 - 2 = 6

부족
```

```text
2^4 - 2 = 14

가능
```

---

### 4단계

CIDR 계산

공식

```text
32 - Host Bit
```

---

영업팀

```text
32 - 7

= /25
```

---

개발팀

```text
32 - 6

= /26
```

---

인사팀

```text
32 - 4

= /28
```

---

## Gateway(GW)란?

Gateway는

```text
다른 네트워크로 나갈 때 사용하는 출입구
```

이다.

보통

```text
Host 범위의 첫 번째 주소
```

를 많이 사용한다.

예)

```text
192.168.1.64/27
```

Host 범위

```text
192.168.1.65

~

192.168.1.94
```

따라서

```text
GW

192.168.1.65
```

로 설정하는 경우가 많다.

※ 반드시 첫 번째 주소를 써야 하는 것은 아니다.

관례적으로 많이 사용하는 것이다.

## 정리

- VLSM은 서로 다른 크기의 Subnet을 만드는 기술이다.
- 필요한 Host 수에 맞게 네트워크를 분할한다.
- IP Address 낭비를 줄일 수 있다.
- 큰 네트워크부터 먼저 분할해야 한다.
- Host 수를 먼저 계산한 뒤 CIDR을 결정한다.
- Network Address, Broadcast Address, Host 범위를 구할 수 있어야 한다.

## 한 줄 요약

VLSM은 필요한 Host 수에 맞춰 서로 다른 크기의 Subnet을 만들어 IP를 효율적으로 사용하는 기술이다.
