# VLSM (Variable Length Subnet Mask)

## VLSM이란?

VLSM(Variable Length Subnet Mask)은 하나의 네트워크를 서로 다른 크기의 Subnet으로 나누는 기술이다.

일반 Subnetting은 모든 Subnet이 동일한 크기를 가지지만, VLSM은 필요한 Host 수에 맞게 서로 다른 크기의 Subnet을 만들 수 있다.

예)

```text
100 Host → 큰 Subnet

50 Host → 중간 Subnet

10 Host → 작은 Subnet
```

따라서 IP Address 낭비를 최소화할 수 있다.

---

## 왜 VLSM을 사용할까?

예를 들어

```text
영업팀 : 100명
개발팀 : 50명
인사팀 : 10명
```

일반 Subnetting을 사용하면 모든 부서에 동일한 크기의 네트워크를 할당해야 한다.

하지만 VLSM은 각 부서에 필요한 만큼만 할당할 수 있다.

```text
영업팀 → /25

개발팀 → /26

인사팀 → /28
```

즉,

```text
필요한 만큼만 나누기
```

가 핵심이다.

---

## VLSM 문제 풀이 순서

```text
1. 필요한 Host 수 확인

2. Host 수가 큰 순서대로 정렬

3. Host Bit 계산
   (2^n - 2)

4. CIDR 계산
   (32 - Host Bit)

5. 큰 네트워크부터 순서대로 할당

6. NA, Host Range, BA 계산
```

---

## 왜 큰 네트워크부터 나눌까?

작은 네트워크부터 나누면 큰 네트워크가 들어갈 공간이 부족해질 수 있다.

따라서 VLSM의 원칙은

```text
큰 것부터 먼저
```

이다.

## Host Bit 계산 예시

| Host 수 | Host Bit | 사용 가능 Host | CIDR |
|----------|----------|----------|----------|
| 100 | 7 | 126 | /25 |
| 50 | 6 | 62 | /26 |
| 30 | 5 | 30 | /27 |
| 10 | 4 | 14 | /28 |

공식

```text
Host

2^n - 2

CIDR

32 - Host Bit
```

## 예제 1

조건

```text
200.200.200.0/24

B팀 : 100 Host
A팀 : 50 Host
C팀 : 10 Host
```

| 팀 | CIDR | Subnet Mask | NA | BA | Host 범위 |
|------|------|------|------|------|------|
| B팀 | /25 | 255.255.255.128 | 200.200.200.0 | 200.200.200.127 | .1 ~ .126 |
| A팀 | /26 | 255.255.255.192 | 200.200.200.128 | 200.200.200.191 | .129 ~ .190 |
| C팀 | /28 | 255.255.255.240 | 200.200.200.192 | 200.200.200.207 | .193 ~ .206 |

## 예제 2

조건

```text
192.168.1.0/24

영업팀 : 50 Host
총무팀 : 30 Host
개발팀 : 12 Host
```

| 부서 | CIDR | Subnet Mask | NA | BA | Host 범위 |
|------|------|------|------|------|------|
| 영업팀 | /26 | 255.255.255.192 | 192.168.1.0 | 192.168.1.63 | .1 ~ .62 |
| 총무팀 | /27 | 255.255.255.224 | 192.168.1.64 | 192.168.1.95 | .65 ~ .94 |
| 개발팀 | /28 | 255.255.255.240 | 192.168.1.96 | 192.168.1.111 | .97 ~ .110 |

## Gateway(GW)

Gateway는 다른 네트워크로 나가기 위한 출입구이다.

보통 Host 범위의 첫 번째 주소를 사용한다.

예)

```text
192.168.1.64/27

Host Range

192.168.1.65 ~ 192.168.1.94
```

일반적으로

```text
GW = 192.168.1.65
```

를 사용한다.

## 암기 포인트

```text
VLSM
= 서로 다른 크기의 Subnet 생성

큰 네트워크부터 분할

Host 계산
= 2^n - 2

CIDR 계산
= 32 - Host Bit

NA
= Network Address

BA
= Broadcast Address

GW
= Default Gateway
```

---

## 정리

- VLSM은 서로 다른 크기의 Subnet을 만드는 기술이다.
- 필요한 Host 수에 맞춰 네트워크를 분할한다.
- 큰 네트워크부터 먼저 분할해야 한다.
- IP Address 낭비를 최소화할 수 있다.
- Host Bit → CIDR → NA/BA 순서로 계산한다.

## 한 줄 요약

VLSM은 필요한 Host 수에 맞춰 서로 다른 크기의 Subnet을 생성하여 IP를 효율적으로 사용하는 기술이다.
