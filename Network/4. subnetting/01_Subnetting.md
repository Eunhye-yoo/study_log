# Subnetting

## Subnetting이란?

Subnetting은 하나의 큰 네트워크를 여러 개의 작은 네트워크로 나누는 기술이다.

Subnet은

```text
Sub + Network
```

의 합성어로,

```text
하나의 큰 Network

↓

여러 개의 작은 Network
```

로 분할하는 것을 의미한다.

Subnetting을 사용하는 이유는 IP Address를 효율적으로 사용하고, Broadcast 범위를 줄여 네트워크를 더 효율적으로 관리하기 위해서이다.

---

## 왜 Subnetting을 사용할까?

예를 들어,

```text
192.168.1.0/24
```

네트워크는 Host를 최대 254개까지 사용할 수 있다.

하지만 실제 환경은

```text
부서 A : 30명
부서 B : 20명
부서 C : 15명
```

처럼 나뉘어 있다.

이때 네트워크를 분리하지 않으면

- IP 낭비
- Broadcast 증가
- 장애 발생 시 영향 범위 증가
- 관리 어려움

문제가 발생한다.

Subnetting은 하나의 큰 네트워크를 여러 개로 나누어 이러한 문제를 해결한다.

---

## CIDR 표기법

CIDR(Classless Inter-Domain Routing)은 Subnet Mask를 간단하게 표현하는 방법이다.

예)

```text
192.168.10.0/24
```

여기서

```text
/24
```

는

```text
255.255.255.0
```

을 의미한다.

자주 사용하는 CIDR

```text
/8  = 255.0.0.0
/16 = 255.255.0.0
/24 = 255.255.255.0
/25 = 255.255.255.128
/26 = 255.255.255.192
/27 = 255.255.255.224
/28 = 255.255.255.240
/29 = 255.255.255.248
/30 = 255.255.255.252
```

---

## IP 주소의 구성

예)

```text
192.168.10.35/24
```

IP 주소는

```text
Network 부분
+
Host 부분
```

으로 구성된다.

Subnetting 문제에서는 아래 3가지를 반드시 찾을 수 있어야 한다.

```text
NA (Network Address)
= 네트워크 자체를 나타내는 주소
= 첫 번째 주소

Host Range
= 실제 장비가 사용할 수 있는 주소 범위

BA (Broadcast Address)
= 같은 네트워크 전체에게 보내는 주소
= 마지막 주소
```

예)

```text
192.168.10.0/24

NA : 192.168.10.0

Host :
192.168.10.1 ~ 192.168.10.254

BA :
192.168.10.255
```

## Host 계산 공식

사용 가능한 Host 수

```text
2^n - 2
```

- n = Host Bit 수
- -2 = NA, BA 제외

예)

```text
/24

Host Bit

32 - 24 = 8

Host

2^8 - 2

= 254
```

---

## Subnet 개수 계산

Subnet 개수 계산 공식

```text
2^n
```

- n = 빌려온 Bit 수

예)

```text
/24

↓

/26
```

Host Bit 2개를 Subnet Bit로 사용

```text
2^2 = 4
```

따라서

```text
4개의 Subnet 생성
```

---

## Block Size(증가값)

시험에서 가장 많이 사용하는 개념 중 하나이다.

공식

```text
256 - 마지막 옥텟 값
```

예)

```text
255.255.255.192

↓

256 - 192

= 64
```

즉

```text
0
64
128
192
```

단위로 Network가 생성된다.

암기하면 매우 편하다.

```text
/25 → 128
/26 → 64
/27 → 32
/28 → 16
/29 → 8
/30 → 4
```

---

## 문제 풀이 순서

Host 수가 주어졌을 때

```text
1. 필요한 Host 수 확인

2. 2^n - 2 계산

3. Host Bit 결정

4. CIDR 계산
   (32 - Host Bit)

5. Subnet Mask 확인

6. NA, Host Range, BA 계산
```

## 예제 1

문제

```text
192.168.53.0/24

Host 50개씩 필요
```

### 풀이

```text
2^5 - 2 = 30 (부족)

2^6 - 2 = 62 (가능)
```

Host Bit

```text
6
```

CIDR

```text
32 - 6

= /26
```

Subnet Mask

```text
255.255.255.192
```

Block Size

```text
256 - 192

= 64
```

결과

```text
Subnet 1

NA : 192.168.53.0
Host : 192.168.53.1 ~ 192.168.53.62
BA : 192.168.53.63

Subnet 2

NA : 192.168.53.64
Host : 192.168.53.65 ~ 192.168.53.126
BA : 192.168.53.127

Subnet 3

NA : 192.168.53.128
Host : 192.168.53.129 ~ 192.168.53.190
BA : 192.168.53.191

Subnet 4

NA : 192.168.53.192
Host : 192.168.53.193 ~ 192.168.53.254
BA : 192.168.53.255
```

총

```text
4개의 Subnet 생성
```

## 정리

- Subnetting은 하나의 큰 네트워크를 여러 개의 작은 네트워크로 나누는 기술이다.
- IP Address 낭비를 줄이고 Broadcast 범위를 감소시킨다.
- Host 계산 공식은 `2^n - 2` 이다.
- Subnet 계산 공식은 `2^n` 이다.
- 문제에서는 NA(Network Address), Host Range, BA(Broadcast Address)를 구할 수 있어야 한다.
- Block Size를 이용하면 Subnet 범위를 빠르게 계산할 수 있다.

## 한 줄 요약

Subnetting은 하나의 큰 네트워크를 여러 개의 작은 네트워크로 분할하여 IP를 효율적으로 사용하고 Broadcast를 줄이는 기술이다.
