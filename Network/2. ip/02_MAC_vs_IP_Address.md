# MAC Address vs IP Address

## 왜 주소가 2개나 필요할까?

네트워크 통신에서는 두 종류의 주소를 사용한다.

- IP Address
- MAC Address

처음 배우면

> "IP 주소만 있으면 되는 거 아닌가?"

라는 의문이 들 수 있다.

하지만 실제 통신에서는 두 주소가 서로 다른 역할을 담당한다.

---

## 한눈에 비교

| 구분 | IP Address | MAC Address |
|--------|--------|--------|
| 의미 | 장비의 위치를 나타내는 주소 | 장비 자체를 식별하는 주소 |
| 종류 | 논리적 주소(Logical Address) | 물리적 주소(Physical Address) |
| 변경 가능 여부 | 변경 가능 | 일반적으로 변경되지 않음 |
| 사용 계층 | Layer 3 (Network) | Layer 2 (Data Link) |
| 예시 | 192.168.0.10 | 00:1A:2B:3C:4D:5E |
| 역할 | 목적지 찾기 | 실제 장비 찾기 |

---

## 비유로 이해하기

택배를 보낸다고 생각해보자.

### IP Address

집 주소

예)

경기도 안양시 ...

---

### MAC Address

집에 사는 사람

예)

홍길동

---

택배기사는

1. 집 주소(IP)를 보고 목적지까지 이동한다.
2. 집에 도착한 후 실제 사람(MAC)에게 택배를 전달한다.

네트워크도 비슷하게 동작한다.

---

## IP Address란?

IP Address는 네트워크 상에서 장비의 위치를 나타내는 주소이다.

쉽게 말하면

"어디에 있는가?"

를 나타낸다.

예)

192.168.0.10

특징

- 변경 가능
- 네트워크 환경에 따라 달라질 수 있음
- 서로 다른 네트워크 간 통신에 사용
- 라우터가 참고하는 주소

---

## MAC Address란?

MAC Address는 랜카드(NIC)에 부여된 고유한 주소이다.

쉽게 말하면

"누구인가?"

를 나타낸다.

예)

00:1A:2B:3C:4D:5E

특징

- 제조 시 부여됨
- 전 세계적으로 고유함
- 일반적으로 변경되지 않음
- 같은 네트워크 내 통신에 사용

---

## 실제 통신에서는?

예를 들어

PC-A

IP : 192.168.0.10

↓

PC-B

IP : 192.168.0.20

으로 Ping을 보낸다고 가정하자.

---

### 1단계

PC-A는 목적지 IP를 확인한다.

```text
192.168.0.20
```

---

### 2단계

하지만 데이터를 보내기 위해서는

상대방의 MAC Address가 필요하다.

현재는 IP만 알고 있는 상태이다.

---

### 3단계

ARP가 동작한다.

```text
192.168.0.20 누구야?

MAC 주소 알려줘!
```

Broadcast 전송

---

### 4단계

PC-B가 응답한다.

```text
내 MAC 주소는

AA-AA-AA-AA-AA-AA
```

---

### 5단계

PC-A는 ARP Cache Table에 저장한다.

```text
192.168.0.20
↓

AA-AA-AA-AA-AA-AA
```

---

### 6단계

이제 실제 통신이 시작된다.

Layer 3에서는

```text
목적지 IP
192.168.0.20
```

를 사용하고

Layer 2에서는

```text
목적지 MAC
AA-AA-AA-AA-AA-AA
```

를 사용하여 Frame을 전송한다.

---

## 왜 IP만으로 통신할 수 없을까?

스위치(Switch)는 IP 주소를 보지 않는다.

스위치는 Layer 2 장비이기 때문에

MAC Address를 보고 동작한다.

따라서 같은 네트워크 안에서 데이터를 전달하려면

반드시 MAC Address가 필요하다.

---

## 왜 MAC만으로 통신할 수 없을까?

MAC Address만으로는

장비가 어느 네트워크에 있는지 알 수 없다.

인터넷처럼 수많은 네트워크를 거쳐 통신하려면

위치를 나타내는 IP Address가 필요하다.

라우터는 IP Address를 보고 경로를 결정한다.

---

## 정리

- IP Address는 장비의 위치를 나타낸다.
- MAC Address는 장비 자체를 식별한다.
- IP는 "어디에 있는가?"
- MAC은 "누구인가?"
- 라우터는 IP를 사용한다.
- 스위치는 MAC을 사용한다.
- 실제 통신에서는 IP와 MAC이 함께 사용된다.
- ARP는 IP Address를 MAC Address로 변환해주는 프로토콜이다.

## 한 줄 요약

IP Address로 목적지를 찾고,

MAC Address로 실제 장비에게 데이터를 전달한다.
