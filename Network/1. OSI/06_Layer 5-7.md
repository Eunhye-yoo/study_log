# OSI 7 Layer (5~7 Layer)

## 전체 큰 그림

OSI 7 Layer는 위로 갈수록 사용자에 가까워지고, 아래로 갈수록 네트워크 장비에 가까워진다.


7 Application → 사용자 서비스
6 Presentation → 데이터 형식 변환
5 Session → 통신 연결 상태 관리


---

## 7 Layer - Application Layer

사용자가 직접 사용하는 서비스가 존재하는 계층이다.

예:
- 웹 브라우저 (Chrome)
- 이메일
- 메신저
- 게임 클라이언트

```text
즉, Application Layer = “프로그램 자체”가 아님
네트워크를 사용하는 "프로그램의 기능 부분"이다.
```
---

## 6 Layer - Presentation Layer

데이터를 서로 이해할 수 있는 형태로 변환하는 계층이다.

주요 역할:
- 데이터 형식 변환 (Encoding / Decoding)
- 암호화 / 복호화
- 압축 / 해제

쉽게 말하면
```text
→ 서로 다른 시스템이 데이터를 이해할 수 있도록 "번역"하는 역할이다.
```
---

## 5 Layer - Session Layer

통신 중인 연결 상태(대화)를 관리하는 계층이다.

주요 역할:
- 연결 시작
- 연결 유지
- 연결 종료
- 통신 재연결

쉽게 말하면
→ "대화가 끊기지 않도록 관리하는 계층"이다.
```text
네트워크 통신은 단순히 1번 주고받고 끝이 아니라:

계속 연결 상태를 유지하거나
끊겼을 때 다시 이어야 함
```
---

## 5~7 Layer 핵심 정리

- Application = 무엇을 할지 (웹, 게임)
- Presentation = 어떻게 표현할지 (형식, 암호화)
- Session = 연결 상태를 어떻게 유지할지

---

## 한 줄 요약

OSI 5~7 Layer는 "사용자 서비스 → 데이터 변환 → 연결 관리" 순서로 동작하는 상위 계층이다.
