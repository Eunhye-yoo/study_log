# UDP (User Datagram Protocol)

## UDP란?

UDP(User Datagram Protocol)는 빠른 데이터 전송을 위한 비신뢰성 통신 프로토콜이다.

즉, 데이터를 "빠르게 전달하지만 정확성은 보장하지 않는 방식"이다.

---

## 왜 UDP가 필요할까?

TCP는 신뢰성을 보장하지만 과정이 많아 느리다.

그래서 다음과 같은 상황에서는 UDP가 더 적합하다.

- 실시간 영상 스트리밍
- 온라인 게임
- 음성 / 영상 통화 (VoIP)
- DNS 질의

---

## UDP 특징

### 비연결형 프로토콜 (Connectionless)

UDP는 통신 전에 연결을 생성하지 않는다.

- 3-way Handshake 없음
- 바로 데이터 전송 가능

---

### 빠른 전송 속도

연결 과정이 없기 때문에 TCP보다 훨씬 빠르다.

---

### 비신뢰성 (Unreliable)

UDP는 다음을 보장하지 않는다.

- 데이터 도착 여부
- 데이터 순서
- 중복 제거
- 재전송

즉, 그냥 보내고 끝이다.

---

### 흐름 제어 없음 (No Flow Control)

수신자의 처리 속도를 고려하지 않는다.

---

### 혼잡 제어 없음 (No Congestion Control)

네트워크 상태를 고려하지 않고 전송한다.

---

## UDP 동작 방식

데이터 생성  
↓  
바로 전송  
↓  
끝

---

## UDP 사용하는 대표 서비스

- 실시간 스트리밍 (IPTV, 라이브 방송)
- 온라인 게임
- DNS (Domain Name System)
- VoIP (음성 / 영상 통화)

---

## TCP vs UDP 비교

| 구분 | TCP | UDP |
|------|-----|-----|
| 연결 | 있음 | 없음 |
| 속도 | 느림 | 빠름 |
| 신뢰성 | 보장 | 없음 |
| 순서 | 보장 | 없음 |
| 재전송 | 있음 | 없음 |
| 흐름 제어 | 있음 | 없음 |
| 혼잡 제어 | 있음 | 없음 |
| 사용 목적 | 정확성 | 속도 |

---

## UDP 핵심 구조

UDP는 최소 기능만 제공하는 프로토콜이다.

```text
최대한 단순하게 → 최대한 빠르게
```

## 핵심 정리
- UDP는 연결을 생성하지 않는다
- 빠르지만 신뢰성을 보장하지 않는다
- 순서, 재전송, 확인이 없다
- 실시간 서비스에 적합하다

## 한 줄 요약

UDP는 "빠르지만 정확성을 보장하지 않는 통신 방식"이다.
